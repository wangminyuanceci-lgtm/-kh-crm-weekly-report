# 全店广告周复盘框架(高管视角)

> 配套脚本:[../scripts/sales_ad_analysis.py](../scripts/sales_ad_analysis.py)
> 数据源:全量 BulkSheetExport + BI数据集 + 销售/库存数据
> 适用读者:**店铺负责人 / 运营总监**
> 适用场景:周一晨会用,5 分钟读完拍 3 个决定
> 配套品线报告:[指定产品分析框架.md](指定产品分析框架.md) (供运营深入)

---

## 设计原则

| 原则 | 体现 |
|------|------|
| **不下钻** | 只到品线级,不展开子ASIN/Campaign |
| **看趋势 + 分布** | 大盘 + 各品线性价比对比,而非具体动作 |
| **异常用色标定位** | ⭐ 性价比好 / ⚠️ 烧多卖少 / 🚨 紧急 / 💀 系统性问题 |
| **决策导向** | 末段必有 3-5 个高管能直接拍的决定 |
| **单页阅读** | 控制在 1.5 页以内,避免管理者翻页 |
| **可下钻** | 末尾给品线详细报告链接,让感兴趣的人深挖 |

## 报告结构(7 节,固定模板)

### 1. 店铺大盘速览
- 一张表:Spend / Sales / 店销 / ACOS / **TACOS** / ROAS / 广告销售占比 / 店铺综合 CVR
- 关键数字加粗,异常加色标
- 一句话整体结论(健康 / 警戒 / 紧急)

### 2. 品线分布矩阵
- 所有品线一张表,按 Spend 倒序
- 必备列:Spend / 占比 / 店销 / 占比 / **性价比标** / ACOS / ROAS / TACOS / 父ASIN 数 / 库存预警
- 性价比判断:
  - ⭐ 性价比好:Spend% < 店销% × 0.7
  - ⚠️ 烧多卖少:Spend% > 店销% × 1.5
  - 平衡:介于之间

### 3. 广告类型分布
- 一行总览:SP / SB / SD 各占 Spend % + ACOS
- 不展开,有异常时一句话点评

### 4. 系统性横向问题
- **库存健康度**:全店 6 档库存父ASIN 数分布(断货/紧急/跟进/健康/偏多/严重过库)
- **自然流量趋势**:广告销售占比是否健康(>85% 偏依赖,需 listing/SEO 介入)
- **跨品线异常品线**:TACOS >15% 或 ACOS >25% 的品线点名

### 5. 本周关键事件(给老板的 3-5 条)
- 升级到管理层的事件(不到这层不要写)
- 例:某主力品线断货 / 某品线广告效率系统性下降 / 某黑洞品线建议砍预算
- 格式:**事件 → 影响 → 建议**

### 6. 决策建议
- 高管能在会议上直接拍的 3-5 个决定
- 不写需要运营自己动手的细节(那些在品线报告里)
- 例:加投 EQUIPMENTS 预算 +30% / 砍 FBA 预算 -20% / 紧急补货 7 个 SKU / 听 GYMNASTICS 详细汇报

### 7. 品线详细报告 — 下钻入口
- 链接所有同期已生成的品线详细诊断报告
- 标注哪些品线"必听汇报"(本周问题最大的 2-3 个)

## 关键阈值(全店级)

| 指标 | 健康 | 警戒 | 紧急 |
|------|------|------|------|
| 全店 TACOS | <12% | 12-18% | >18% |
| 全店 ROAS | >5 | 3-5 | <3 |
| 广告销售占比 | <70% | 70-85% | >85%(过度依赖广告) |
| 店铺综合 CVR | >4% | 2-4% | <2% |
| 严重过库父ASIN 占比 | <20% | 20-40% | >40%(库存灾难) |
| 单品线 TACOS | <12% | 12-18% | >18%(广告依赖严重) |
| 单品线 ACOS | <20% | 20-30% | >30% |

## 周复盘 vs 品线诊断 — 内容边界

| 维度 | 全店周复盘 | 品线详细诊断 |
|------|---------|------------|
| 读者 | 店铺负责人 | 品线运营 |
| 篇幅 | 1.5 页 | 3-5 页 |
| 下钻深度 | 品线级停 | 子ASIN级 |
| 案例数 | 0(只看分布) | 5+ 个父ASIN拆解 |
| Campaign 动作 | 不展开 | 详细列 P0-P3 |
| 决策颗粒度 | 战略级(加投/减投/补货) | 战术级(改主图/加 Negative/降 CPC) |
| 生成频率 | 每周一份 | 重点品线每周,其他双周 |

## 周复盘输出位置

```
AmazonAnalysisResults/<日期>/
├── 店铺广告周复盘-<日期>.md         ← 周复盘(本框架产物)
├── 产品销售广告情况.xlsx              ← 全店明细(供下钻)
├── 产品线广告数据基准值.xlsx
├── 该品类的广告内容.xlsx
├── 未关联产品信息的广告.xlsx
├── GYMNASTICS/
│   └── 广告分析报告-GYMNASTICS-<日期>.md   ← 品线详细
├── WEIGHTS/
│   └── 广告分析报告-WEIGHTS-<日期>.md
└── ...
```

---

## 周复盘生成流程

1. 跑 `python scripts/sales_ad_analysis.py --output-dir <日期>`(不指定 product-lines = 全量)
2. 读取 `产品线广告数据基准值.xlsx`(品线整体基准 sheet)和 `产品销售广告情况.xlsx`(父ASIN级 sheet)
3. 按本框架的 7 节模板填表
4. 对本周问题最大的 2-3 个品线(通常是: 烧多卖少 / 库存系统性问题 / TACOS 突破警戒线), **同步出品线详细诊断报告**, 在第 7 节给出链接
5. 周复盘控制在 1.5 页内, 把细节留给品线报告

## 框架与品线诊断框架的协同关系

| 关系 | 说明 |
|------|------|
| **数据基底相同** | 都来自 `sales_ad_analysis.py` 的输出 |
| **粒度互补** | 周复盘给方向, 品线诊断给执行 |
| **结论闭环** | 周复盘"建议听 X 品线汇报"→ 老板看 X 品线详细诊断报告 → 决策落地 |
| **避免重复** | 周复盘里 1 行的内容, 品线诊断里展开成一段 |
