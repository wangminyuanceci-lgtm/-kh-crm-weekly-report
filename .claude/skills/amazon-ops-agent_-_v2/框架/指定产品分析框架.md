# 亚马逊指定产品分析框架

> 配套脚本:
> - [scripts/sales_ad_analysis.py](../scripts/sales_ad_analysis.py) — Layer 1-5 销售广告综合分析(用于下钻部分)
> - [scripts/trend_layer.py](../scripts/trend_layer.py) — 4 周趋势计算(概览主体)
> - [scripts/target_layer.py](../scripts/target_layer.py) — 月度目标达成(销售额 + 销量)
> - [scripts/promotion_layer.py](../scripts/promotion_layer.py) — 促销活动 + 折扣影响

> 适用场景:用户指定一个或多个产品(品线/父ASIN/SKU 任一粒度),做指定时间段的运营诊断 + 4 周趋势对照 + 落到具体动作

---

## 0. 指定产品的三种粒度

| 粒度 | 用户表达示例 | 报告主体 | 是否要展示所在品线占比 |
|------|------|------|:---:|
| **品线** | "分析 EQUIPMENTS 品线" | 该品线所有父ASIN | ❌(本身就是品线)|
| **父ASIN** | "分析 B0GK8NKCDJ" / "B0GK8NKCDJ + B0GK7CBJ6G" | 这几个父ASIN | ✅ |
| **SKU** | "分析 RF-BENCH-BWB01BLK" | 这几个 SKU | ✅ |
| 不指定 | 没说 → 走"全品类周报" | 全店 | ❌ |

**关键原则**:**报告主体永远是用户指定的产品**,不是"所在品线大盘"。所在品线只作为"占比参照"在第 2 节出现一行,不展开。

## 1. 时间范围说明(报告必含,放在最前)

每份报告都要清楚列 3 个时间:

| 范围 | 说明 |
|------|------|
| **本期** | 用户指定的 `--start-date` ~ `--end-date` |
| **最近 4 周** | 按 end_date 往前推 4 个 7 天: W4=本期 / W3=前7天 / W2=前14天 / W1=前21天 |
| **目标月** | end_date 所在月(默认);跨月默认按 end_month 模式,跨月时显式说明 |

## 2. 报告四部分结构

```

│ ① 概览:指定产品的 4 周趋势(核心 6 类指标)     │  ← 主体
```

## ① 概览:指定产品的 4 周关键指标趋势

每个指定产品(父ASIN 或 SKU)一张表,**带品名**,展示 4 周趋势 + 本期目标达成。

### 必含 6 类指标

| 指标 | 单位 | 来源字段 | 解读重点 |
|------|------|------|---------|
| **销售额(折后)** | $ | sales 表 `实收销售额`(=`净销售额`) | 主营收 |
| **销量** | 件 | sales 表 `销量` | 主出货量 |
| **流量** | Sessions | sales 表 `Sessions` | 自然/广告引流总和 |
| **转化** | % | 订单量 / Sessions | 流量质量 |
| **客单价(折前)** | $ | 商品销售额 / 订单量 | 售价水平 |
| **客单价(折后)** | $ | 实收销售额 / 订单量 | 实际客单价 |

### 表格模板

```
### 产品 X:[父ASIN/SKU 编号] — [品名] (品线 / 产品线)

| 指标         | W1(4/29-5/5) | W2(5/6-5/12) | W3(5/13-5/19) | W4(5/20-5/26)本期 | W4 vs W3 |
|--------------|-------------:|-------------:|--------------:|------------------:|---------:|
| 销售额(折后) |       $X,XXX |       $X,XXX |        $X,XXX |            $X,XXX |    +/-X% |
| 销量         |         XX 件 |         XX 件 |          XX 件 |              XX 件 |    +/-X% |
| Sessions     |        X,XXX |        X,XXX |         X,XXX |             X,XXX |    +/-X% |
| 转化率       |         X.X% |         X.X% |          X.X% |              X.X% |     +Xpp |
| 客单价(折前) |       $XXX.XX |       $XXX.XX |        $XXX.XX |            $XXX.XX |    +/-X% |
| 客单价(折后) |       $XXX.XX |       $XXX.XX |        $XXX.XX |            $XXX.XX |    +/-X% |
| 折扣率       |          X.X% |          X.X% |           X.X% |               X.X% |     +Xpp |

**本月目标达成**(截至 W4 end_date):
- 销售额: ✅/⚠️/🔴  $XX,XXX / $XX,XXX = XX% (时间进度 XX%)
- 销量  : ✅/⚠️/🔴  XX 件 / XX 件 = XX% (时间进度 XX%)

**4 周诊断**(一句话):
- "持续上升""持续下滑""突然反转""高位震荡""转化率塌方"等模式
```

### 4 周趋势诊断的 6 种模式

| 趋势模式 | 数据特征 | 处理 |
|---|---|---|
| **持续上升** | 4 周连续 +5%~+20%/周 | 找原因复制,加投/守库 |
| **持续下滑** | 4 周连续 -5%~-20%/周 | 警惕长期问题,从 Sessions/转化率 找根因 |
| **突然反转** | 前 3 周平稳,W4 突然 +/-30%+ | 找单点原因:活动 / 断货 / 排名变化 / 季节 |
| **高位震荡** | 4 周间波动 ±20% 但无趋势 | 关注供应链/广告投放节奏稳定性 |
| **转化率塌方** | Sessions 稳但转化率 < 上周 × 0.5 | listing / 价格 / 库存提示符 / 评论新增问题 |
| **客单价分化** | 折前涨 + 折后跌 | 折扣加大但价格基础没动 → 看是否赔本促销 |

### 4 周趋势 vs 单期同环比的区别

- **同环比**(本期 vs 单一对比期):点对点,容易被单周噪声误导
- **4 周趋势**(本期 + 前 3 周):**线性趋势**,看走向,**周报推荐用此**

## ② 所在品线占比(仅父ASIN/SKU 粒度时有)

只需要 **一行 + 一句话**,不要展开成"品线大盘分析"。

```
> 所在品线 [EQUIPMENTS]:
> - 本期品线总销售额 $XX,XXX
> - 本产品销售额 $X,XXX 占品线 X.X%
> - 性价比标签:⭐ 性价比好(店销占比 > Spend 占比 × 1.3)/ 平衡 / ⚠️ 烧多卖少
```

如果指定多个父ASIN/SKU 都属同品线,合并算一次品线总额;否则每个产品都给一行。

## ③ 期内促销活动 + 折扣影响

**数据源**:`亚马逊美国站促销活动` 表

**取数关键**:活动期与查询期"**时间重叠**"(不是简单的开始时间在范围内):
- `开始时间 <= end_date AND 结束时间 >= start_date`

### 必含两类输出

**A. 活动列表**(按 `促销ID` 聚合,带覆盖的子SKU)

| 类型 | 内部描述 | 起止 | 状态 | 覆盖 SKU |
|------|------|------|------|---------|
| 秒杀/优惠券/价格折扣 | xxx | mm/dd-mm/dd | 已结束/进行中 | 列表 |

**B. 折扣对销售的影响**(从 sales 表 `折扣金额` / `商品销售额` / `实收销售额` 直接汇总)

- 商品销售额(折前) → 实收销售额(折后) → 折扣金额 → **折扣率 = 折扣金额 / 商品销售额**
- 折前客单价 → 折后客单价 → **客单价下挫 %**
- 订单量

### 6 种促销诊断模式

| 现象 | 诊断 | 处理 |
|---|---|---|
| W4 销售 +X% **且** W4 有促销 + W4 折扣率 > W3 | **促销驱动增长**(可能促销后回落) | 别用本期数据作"加投放量"依据 |
| W4 销售 +X% **且** W4 有促销 + W4 折扣率 < W3 | ⭐ **真自然增长**(少打折反而卖更多)| 守住,可继续 |
| W4 销售 -X% **且** W4 有促销 | **促销也救不回**(产品力/外部冲击) | P0 排查 |
| 折前客单价稳 + 折后客单价跌 | 折扣力度加大但价格基础没动 | 看是否吃利润但没换销售 |
| 折扣率 >20% **且** 销售只小涨 | 烧钱促销效率低 | 重审 Coupon/Deal 策略 |
| 折扣率 <2% **且** 销售环比大跌 | 促销很弱也没拉动 | 看 listing/SEO,可能自然流量塌方 |

## ④ 下钻分析(动作落地)

下钻分两层:**父ASIN 粒度 → 拆子SKU**;**SKU 粒度 → 拆 Campaign × SKU + 三因素归因 + LinkFox 验证**。

下钻**只看本期快照**(不算 4 周趋势),除非要做单 SKU 的根因诊断(此时拉广告/促销/排名三层数据看变化)。

### 父ASIN 粒度的下钻 = 子SKU 拆解(必带子ASIN)

每个 SKU 必须显示对应的**子ASIN**,因为运营操作时要按子ASIN 在后台找。

| SKU | **子ASIN** | 品名 | 本期销售 | Sessions | 店CVR | **Spend** | Spend/组占 | **广告Orders** | **ACOS** | **TACOS** | **ROAS** | 库存6档 | 期内活动? |
|---|---|---|---:|---:|---:|---:|---:|---:|---:|---:|---:|---|:---:|
| RF-XXX | B0XXX | 品名 | $X,XXX | XXX | X.X% | $XXX | XX% | X | XX% | XX% | X.X | 🟢/🟡/🔴 | ✅/— |

**列字段说明**:
- 子ASIN: sales 表 `子ASIN` 字段
- Sessions / 店CVR: sales 表
- Spend / 广告Orders / ACOS / ROAS: BulkSheet 子SKU 级
- TACOS = Spend / sales 表 `实收销售额`
- 期内活动: 来自 promotion 表(✅ 有 / — 无)

**重点动**:占组 Spend ≥10% 且有异常的 SKU + 具体动作
**知悉不动**:占比小但有故事的 SKU(短一行)

### SKU 粒度的下钻 = 三因素根因诊断(广告 + 促销 + ASIN 排名)

针对"必须深入"的单 SKU(销售剧烈变化、转化率塌方等),拉 4-6 周的**三类数据**对照,找出"销售变化的真正原因":

#### 必拉的三层数据

**① 广告趋势(BI 表,4-6 周)**

| 周 | Spend | Orders | Sales | Clicks | Impressions | ACOS | ROAS | CPC | 广CVR |
|---|---:|---:|---:|---:|---:|---:|---:|---:|---:|

观察:Spend / Impressions / 广告Orders 跟销售变化是否同步,**强相关 → 广告是主因**。

**② 促销活动(promotion 表,期内重叠)**

| 类型 | 描述 | 起止 | 折扣价 | 涉及 SKU |
|---|---|---|---:|---|

观察:促销期是否覆盖销售爆点/低谷期;折扣率变化是否能解释销售变化。

**③ ASIN 排名/流量入口(LinkFox)— 必须调用**

> **调用范围规则**(避免每个子ASIN 都调,LinkFox 1-5 分钟/次,成本敏感):
>
> | 指定产品粒度 | 调用范围 |
> |---|---|
> | **品线** | 该品线下 Spend Top 5 父ASIN,每个父ASIN 选 Top 3 子ASIN(按销售/Spend 综合)|
> | **父ASIN(单个或多个)** | 每个父ASIN 选 **Top 3 子ASIN**(按销售 + Spend 综合,去重) |
> | **SKU(单个或多个)** | 直接对指定 SKU 对应的子ASIN 调 |
>
> **Top 3 选法**:取销售额 Top 3 和 Spend Top 3 的并集(通常 3-5 个,优先动的就是这些)。
> 不展示"未选中"子ASIN 的 LinkFox 数据时,在报告里**显式声明"已对 Top X 子ASIN 调 LinkFox,其他子ASIN 因占组小未调,如需可补"**。

通过 [Linkfox_Agent_skill](../../.claude/skills/Linkfox_Agent_skill/) 对每个目标子ASIN 调用:
1. `@Keepa-亚马逊-商品详情` — BSR / 评分 / 评论数 / 价格历史
2. `@SIF-ASIN的关键词` — 主关键词自然排名(可选,根据时间预算)
3. `@SIF-ASIN流量来源` — **流量来源结构(自然/SP/SB/视频/AC)+ 本周期 vs 上周期对比** ⭐ 最关键
4. `@亚马逊-商品评论(美国站)` — 最近 1-2 星差评

关注:
- 流量关键词总数 / 总曝光分数 本周期 vs 上周期变化
- SP/SB/SD/视频广告曝光各自变化(印证广告侧 ② 拉的数据)
- 自然搜索曝光是否稳定(如果稳定 → 自然流量没问题,广告是原因)
- Amazon's Choice 标记进出数量(广告砍多了会导致 AC 标记丢失,反过来又砍流量,是个负循环)
- 近期差评(看转化率塌方是否有产品力原因)

#### 三因素归因表(必含,放在 SKU 根因诊断报告最末)

| 因素 | 关键数据 | 与销售变化相关度 |
|---|---|:---:|
| **① 广告变化** | Spend X% / Impressions X% / 广告 Orders X% | 🔴 强相关 / 🟡 中等 / 🟢 弱相关 |
| **② 促销变化** | 折扣率 X% → Y% / 是否新增活动 | 🔴 / 🟡 / 🟢 |
| **③ ASIN 排名变化** | 流量关键词 X→Y / SP 曝光 X% / AC 标记进出 | 🔴 / 🟡 / 🟢 |

一句话归因结论:**"销售下滑主因是 {广告/促销/排名/listing/差评/其他} 变化"**

#### SKU 粒度也要做 Campaign × SKU 级动作

按 Campaign 类型分桶(本品/Auto/品牌词/拓流/防御),只看本期 Spend/Sales/ACOS,**单周不下 Pause 强结论**(纪律 #17)。

**异常 Campaign 排查**:个别异常 Campaign(如品牌词 0 转化)可用 BI 表/广告位报告拉 4-8 周日级趋势作为工单依据,但不进周报正文。

### 库存 6 档(替代单一可售天数)

| 标签 | 触发(件数 + 可售天数)| 含义 |
|---|---|---|
| 🚨 断货 | 库存=0 或 <5 件 | 必须停广告 + 补货 |
| 🔴 紧急补货 | < 30 天 | 跟进补 + 暂停广告避免引流空 |
| 🟡 跟进补货 | 30-60 天 | 加入补货计划 |
| 🟢 健康 | 60-200 天 | 正常运营 |
| ⚠️ 库存偏多 | 200-500 天 | 考虑加投清库 |
| 💀 严重过库 | > 500 天 | "清库存导向":加投/降价/Coupon,不停广告 |

### 库存数据交叉验证纪律

**库存表显示 0 + 本期销售有订单 → 大概率是数据失真**(在途未计入/时点错位),不能凭此停广告。真断货标准 = 库存表 0 **且** 本期销售 0 **且** Amazon 后台 Available=0。

### Campaign 类型分桶 + 决策原则

| 类型 | 典型命名 | 决策原则 |
|---|---|---|
| **本品** | "本品""own product" | ACOS 应最低;异常高 → 出价过高或关键词污染 |
| **Auto** | "auto""自动" | 0 转化 + 高花费 → 暂停转手动 |
| **品牌词** | "品牌词""brand" | 应是最高 ROI;ACOS 异常 → 怀疑竞品截胡 |
| **类目防御/捡漏** | "捡漏""防御" | 测试性质,单条不痛但要监控合计 |
| **SD 本品防御** | "防御 SD""VCPM" | 防御性质,小预算可接受 |
| **手动精准** | "手动" + 关键词命名 | 重点维护,效率最高 |

## 关键纪律(贯穿整个框架)

1. **报告主体永远是指定产品**,不是大盘 — 大盘只在第 2 节作"占比参照"出现一行
2. **概览用 4 周趋势**,不用单期同环比(避免单周噪声)
3. **客单价必须两套**(折前 + 折后),单一客单价会被促销误导
4. **库存 0 + 销售非 0 → 先核实再动**(纪律 #16)
5. **单周 Campaign 数据不下 Pause 强结论**(纪律 #17),要看 4-9 周趋势
6. **SD/SB 不直接停**,先排查投放对象(本品防御/竞品/VCPM/Auto)
7. **品牌词必须健康**,品牌词 0 转化或 ACOS 突涨必须 P1 排查
8. **目标达成销售额 + 销量 两套**,销量达成低于销售额达成 = 卖高价款多
9. **跨月默认按 end_month**;周报跨月时报告里加一句说明
10. **同比 NaN ≠ 异常**(新品多),留空不要硬填 0
11. **促销 SKU 涉及的销售要标识**(增长是促销驱动还是真增长,影响下周决策)

## 使用流程

1. **指定产品**:用户给品线/父ASIN/SKU(或不指定 → 全品类)
2. **生成 4 周趋势**:`trend_layer.compute_trend(end_date, weeks=4, target_*)`
3. **目标达成**:`target_layer.compute_target_achievement(end_date, target_*)`
4. **促销活动**:`promotion_layer.analyze_promotions(start, end, target_*)`
5. **下钻**:跑 `sales_ad_analysis.py --compare wow` 拿子SKU 级 + Campaign 级数据(仅对"必须深入"的产品)
6. **组装报告**:按四部分结构输出 Markdown
