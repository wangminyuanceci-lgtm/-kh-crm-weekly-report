## 动作字段标准

| 字段 | 必填 | 规范 |
|---|---|---|
| **ID** | ✅ | A001 起递增,本周内全局唯一 |
| **状态标签** | ✅ | `[待执行]` / `[已执行]` / `[已验证]` / `[已取消]` |
| **标题** | ✅ | 一句话说清"对什么做什么",如"RACKS-B0XXX 暂停 SP-Auto" |
| 优先级 | ✅ | P0(本周必做)/ P1(本周尽量)/ P2(下周也行) |
| 品线/父ASIN | ✅ | 准确到父ASIN |
| 动作类型 | ✅ | 暂停广告 / 加预算 / 改 Bid / 改关键词 / 促销 / 改 Listing / 补货 / 防御性 SD 等 |
| 根因 | ✅ | 1-2 句话,带数据;若用了 linkfox 标注来源 |
| 预期效果 | ✅ | 具体数字,如"ACOS 降到 <100%, 节省 $120/周" |
| 执行人 | ✅ | 谁来做(投手 / 运营 / 美工 / 采购) |
| 计划完成 | ✅ | 具体日期 |
| 实际执行 | ⚠️ 执行后填 | 实际完成日期 + 1 句话备注 |
| 复查节点 | ✅ | 下次哪份周报看什么指标 |
| 复查结论 | ⚠️ 验证后填 | ✅ 达预期 / ⚠️ 未达 / 🔴 反向恶化 |

## 状态流转

```
[待执行]
   ↓ 运营完成执行 → 手动改文件
[已执行]
   ↓ 下周 /weekly-review 自动回读 → 对比指标
[已验证] (达预期)  或  追加新动作(未达/反向)
   ↓ 不再出现在下周复查表
归档
```

**`[已取消]`** 的场景:
- 情况变化(如本周突然要冲 PD 大促,P2 动作让位)
- 执行后发现根因错了,动作不再合理
- 连续 2 周 `[待执行]` 没人接 → 自动标 `[已取消]`,在复查表里点名

## 写入时机

| 时机 | 触发者 | 动作 |
|---|---|---|
| `/weekly-review` 跑完 | Agent | 新建 `actions/YYYY-WNN.md` + 写入本周诊断产出的动作(A001 起) |
| 用户下钻后 Agent 给建议 | Agent | 追加到本周文件,ID 从下一个未用号开始 |
| 运营线下执行完 | 运营手动 | 把状态从 `[待执行]` → `[已执行]`,填实际执行 + 备注 |
| 下周 `/weekly-review` 复查 | Agent | 把状态从 `[已执行]` → `[已验证]`,填复查结论 |

## 复查规则(下周 Agent 自动跑)

`/weekly-review` Step 2 自动读上周 actions:

### 对 `[已执行]` 未验证 的动作

1. 根据"复查节点"字段提取要看的指标
2. 从本周脚本输出的 reports/ 中提取对应数据
3. 与"预期效果"对比 → 填 3 类结论:

| 结论 | 判定 | 后续 |
|---|---|---|
| ✅ **达预期** | 实际 ≥ 预期 80% | 状态改 `[已验证]`,关闭 |
| ⚠️ **未达预期** | 实际 < 预期 80% 但方向正确 | 维持 `[已执行]`,复查节点延 1 周;若 2 周仍未达,触发"原因分析"下钻 |
| 🔴 **反向恶化** | 指标向反方向走 | 立即在本周报告 A 段标记 + 产出新动作(回滚或调整) |

### 对 `[待执行]` 未执行 的动作

1. 若计划完成日已过 → 标 `[已逾期]`
2. 连续 2 周逾期 → 自动标 `[已取消]` 并在本周报告 A 段点名

## 动作类型 → 复查指标对照

| 动作类型 | 复查指标 | 复查周期 |
|---|---|---|
| 暂停广告 | Campaign Spend(应=0)+ 品线总 Spend 下降 + ACOS 改善 | 1 周即可 |
| 加预算 | Campaign Spend(应上升)+ Sales(应上升)+ ROAS(应稳定) | 1-2 周 |
| 改 Bid | CPC 变化 + Impression Share + ACOS | 1 周 |
| 改/加关键词 | 新词出 Impressions + 老词分流 | 2 周 |
| 加 Coupon | Sessions(应 +20-40%)+ CVR(应 +3-5pp)+ 销量倍数 | 1 周(立刻见效) |
| Lightning Deal | Deal 期间销量倍数 + Deal 后 1 周 BSR 是否维持 | 2 周 |
| 改 Listing(主图/标题/A+) | CTR + CVR + Sessions | 2-3 周 |
| 补货 | 库存可售天数恢复 + 排名恢复 | 2-4 周 |
| 防御性 SD | 本品页面竞品占位减少 + 同期销售稳定 | 2-4 周(滞后) |

**复查节点写法示例**:`W22 周报检查 ACOS 是否回落到 <100%`(明确节点 + 明确指标 + 明确阈值)。

## 不要做的事

- ❌ **不要写没有复查节点的动作** — 没法验证 = 永远闭不了环
- ❌ **不要写"优化 XX"这种模糊动作** — 必须能落到具体后台操作
- ❌ **不要本周追加超过 15 条动作** — 运营做不完,P0+P1 控在 5-8 条
- ❌ **不要把"持续监控"当动作** — 监控不是动作,是周复盘本身的工作
- ❌ **不要等待运营反馈再写复查** — 自动从数据读,数据不会撒谎
