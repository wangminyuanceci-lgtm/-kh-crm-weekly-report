# 自助下钻 SOP — 提问路由表

> 触发:运营在对话框提任何非周报的问题
> 目的:把模糊的运营问题精准路由到对应 SOP / 报告 / linkfox skill,避免散乱回答

---

## 核心原则

1. **先识别问题类型** → 再决定调用哪个 SOP
2. **必读最新一周报告** → 在 `reports/` 下找最近 YYYY-WNN/,先看脚本产出的数据再回答
3. **归因落地到动作** → 每次下钻都产出 1-N 条新动作,**自动追加**到本周 `actions/YYYY-WNN.md`
4. **金额门槛**:对 Spend < $100/周 或占组内 < 10% 的项目,不出动作建议(钱小不值得动)

## 路由不匹配时的兜底

如果用户问题没匹配上面任何一条:

1. **澄清问题**:用 1-2 个问题缩小范围(例:"你是想看广告效率,还是销量趋势,还是排名?")
2. **直接查数据**:读 `reports/最新周/` 下的 .xlsx 找数据,先给数据再说判断
3. **不要编故事**:如果数据看不出来,告诉用户"现有数据看不到 X,需要调 linkfox / 查 Amazon 后台 / 补充 Y 数据"

---

## 下钻输出归档

**每次下钻完都要做**(不要忘):

1. 在回答里给运营**3 步内可执行**的动作
2. 用 [模板/动作清单模板.md](../模板/动作清单模板.md) 格式追加到 `actions/YYYY-WNN.md`
3. 动作 ID 从该文件下一个未用号开始(如已有 A001-A005,新加 A006)
4. 状态设为 `[待执行]`,复查节点设为 W+1 或 W+2 周报

详细规范见 [动作追踪SOP.md](动作追踪SOP.md)。
