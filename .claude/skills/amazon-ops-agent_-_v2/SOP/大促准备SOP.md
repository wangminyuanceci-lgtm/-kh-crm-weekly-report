# 大促准备 SOP（Prime Day / BFCM / 大型节日）

> 适用场景：Amazon 大型促销活动前 8 周到大促后 2 周
> 配套：[投手周复盘SOP.md](投手周复盘SOP.md) + [品牌与销量增长SOP.md](品牌与销量增长SOP.md)

---

## 🗓️ 时间倒推总览

| 时点 | 阶段 | 核心任务 |
|---|---|---|
| **T-8 周** | 战略规划 | 选 SKU + 申请 Prime Day Deal |
| **T-6 周** | 库存备货 | 发 FBA 头程 |
| **T-4 周** | Listing + 评论 | 优化主图/A+ + Vine 送评 |
| **T-2 周** | 广告预热 | 提预算 + 关键词布局 |
| **T-1 周** | 促销设置 | Coupon / PED + 最终库存确认 |
| **T-0** | 大促当天 | 每 2h 实时监控 |
| **T+1-2 周** | 收割 + 复盘 | View-through 转化 + 数据分析 |

## 🎯 T-8 周：战略规划

### A. 选参战 SKU

| 优先选 | 不要选 |
|---|---|
| 高 BSR 排名（Top 1000） | 库存不足 |
| 评分 ≥ 4.3 + 评论 ≥ 50 | 评分 < 4.0 |
| 高库存（可备到 90d+） | 新品（评论太少）|
| 毛利率 ≥ 30% | 毛利薄 < 20% |

### B. 申请 Deal

| Deal 类型 | 时长 | 流量 | 申请节点 |
|---|---|---|---|
| **Prime Day Deal**（最强） | 4-12h | 5-10 倍 | T-8 周邀请制 |
| Lightning Deal | 6h | 3-5 倍 | T-4 周可申 |
| 7-Day Deal / Best Deal | 1 周 | 2-3 倍 | T-4 周 |
| Coupon（绿标） | 自定 | 转化提升 20-40% | T-1 周即可 |

**关键**：Prime Day Deal 是 Amazon 邀请制 — 大卖家自动收邀请，未收到的找客户经理 / Vendor Manager 申请。

## 📦 T-6 周：库存备货（最容易踩的坑）

### 备货量公式

```
PD 当天预估销量 = 平时日均 × 5（流量 5 倍）× 2 天
PD 前后 2 周长尾 = 平时 × 2 × 14 天
安全库存 = 平时日均 × 60d

总备货 ≥ (5x × 2) + (2x × 14) + 60d 安全
```

**例**：平时日均 10 单 → 总备货 ≥ 980 件

### FBA 头程时间表

| 时点 | 动作 |
|---|---|
| T-6 周 | 头程发出（海运 4-5 周）|
| T-4 周 | 头程到达 + Amazon 入库 1-2 周 |
| **T-2 周** | **必须全部 Available** |
| T-1 周 | 不再上货（来不及入库）|

### 🚫 库存红线

- T-2 周主力 SKU 库存 < 60d → **立即停 PD 报名**
- 缺货 1 周 → 排名掉 30-50 位 → 影响 3-6 个月

## ✏️ T-4 周：Listing 优化 + 抢评论

### Listing 关键动作

| 优化点 | 怎么做 | 目的 |
|---|---|---|
| **主图加 Deal 元素** | 角落加 "Save XX%" 视觉 | PD CTR 提升 |
| **标题埋大促词** | 末尾加 "Prime Day Special" | 季节词捕获 |
| **A+ Content** | 加促销信息板块 | 提升转化 |
| **Brand Story** | 突出品牌信任 | 新客转化 |
| **视频/3D 主图** | 升级 listing 转化率 | 直接提升 CVR |

### 抢评论（**Vine 必做**）

- 主力 SKU 主动启动 Vine 送评 30 单
- 评论数从 100 → 200 → CVR 提升约 15-20%
- T-4 周启动来得及（Vine 4-6 周到位）

## 🎯 T-2 周：广告预热（**最关键**）

### 为什么提前 2 周加广告

用户购买决策路径：
```
PD 前 7-14 天: 看广告 → 加购物车
PD 当天: 从加购车直接下单
```

**PD 前 2 周加广告 = 给 PD 当天填充加购车**。

只在 PD 当天加广告，用户没研究过你的产品，CVR 远不如已加购车的。

### 战术调整

| 维度 | 平时 | T-2 周 → PD 期 |
|---|---|---|
| 主力 SKU 日预算 | $50 | **$200-300**（4-6 倍） |
| Top of Search Bid+ | +100% | **+300% ~ +500%** |
| 核心关键词 Bid | 健康水平 | 提 30-50% |
| Coupon 准备 | - | T-1 周设好 15-20% |
| 新建 PD 专属 Campaign | - | 投季节词 |
| SD 浏览再营销 | - | T-1 周启动 |

### 关键词布局

**新建 SP Exact Campaign 投季节词**：
```
- prime day deals [品类]    例: prime day deals dumbbells
- prime day [核心词]         例: prime day weight plates
- amazon prime day [品类]
- best prime day [品类] deals
```

**预算**：$10-20/天 × 5-10 个新季节词 = $50-200/天

## 🎟️ T-1 周：促销最终设置

### 必做促销组合

| 工具 | 怎么设 | 重点 |
|---|---|---|
| **Prime Exclusive Discount (PED)** | 卖家中心 → Promotions → PED | **必做** — 只对 Prime 会员显示 |
| **Coupon 15-20%** | 同 Coupon SOP | 配合 PED 双重折扣感 |
| **Deal 时段确认** | 后台查 Lightning Deal 排期 | 避免冲突 |
| **Free Shipping** | FBA 自动 | - |

### 价格策略（合规重要）

**Amazon 要求**：Deal 价必须比过去 30 天最低价**低 5%+**

操作：
- T-4 周：原价小幅涨价 5-8%（建立"高基准价"）
- T-2 周：维持原价
- T-0：Deal 价降到目标价

⚠️ 不能"突然降价"否则 Deal 被拒。

### 后端准备

- Buyer Messages 团队加班排班
- 退货处理流程确认
- 售后 FAQ 模板准备

## 🔥 T-0 大促当天：实时监控（每 2h 看一次）

### 监控指标

| 指标 | 频率 | 异常处理 |
|---|---|---|
| **库存余量** | 每 2h | 降到 30d → 立即停广告防超卖 |
| **BSR 排名** | 每 4h | 突破历史最高 → 加预算 |
| **广告 Daily Budget** | 每 2h | 用完追加 50-100% |
| **CVR** | 每 4h | 突降 → 检查 Listing/价格 |
| **竞品价格** | 每 4h | 竞品降价 → 决策跟随否 |
| **差评涌入** | 每 4h | 异常多 → 看产品问题 |

### 当天时间表

```
00:00 PD 启动
- 确认 Deal/Coupon/PED 正常显示
- 确认广告预算充足
- 监控前 3h 数据（最关键）

08:00 北美工作日早晨
- 看排名/销量是否突破
- 必要时追加预算

12:00 北美午餐高峰
- 第二个销量峰值
- 看竞品价格

18:00 北美晚饭后
- 第三个销量峰值
- 关注库存余量

次日 PD Day 2
- 复盘 Day 1 数据
- 调整 Day 2 策略
```

## 📊 T+1-2 周：收割 + 复盘

### View-through 收割期

PD 期间大量用户"看了没买" → PD 后 1-2 周回来下单。

**战术**：
- 维持 **SD 浏览再营销**（捕获 view-through 用户）
- 不要立刻砍广告预算 — 收割期 ROI 极高
- Coupon 改温和 5-10%（不继续大促力度）

### 复盘指标

| 维度 | 看什么 |
|---|---|
| 销量 | PD 2 天 vs 平时 2 天 — 倍数 |
| ACOS | PD 期 ACOS 15-25% 正常 |
| 库存 | 有没有断货？多久？ |
| Listing | CTR / CVR 是否提升 |
| **品牌词搜索量** | PD 后是否上升（长期价值）|
| 评论 | PD 期间评论数增加多少 |

## 💡 经验法则

### 1. **T-2 周库存不够立即停 PD 报名**

继续报 PD 会缺货 → 排名死亡 → 6 个月恢复期。损失远大于"少做这次 PD"。

### 2. **PD 当天 Top of Search 必拉满**

PD 流量爆发 5-10 倍，Top of Search 是搜索结果第一位置 → 黄金中的黄金。Bid+ 即使 +500% 也值得。

### 3. **价格合规提前 30 天准备**

Amazon Deal 价必须比过去 30 天最低价低 5%+。临时降价会被拒，要提前 30 天就涨上去。

### 4. **PD 不仅是销量，更是品牌建设**

PD 期间品牌词搜索量增长 30-50%，**真正的长期价值**是接下来 3-6 个月自然流量增加 → TACOS 下降。

### 5. **PD 后的 view-through 收割期不能砍预算**

PD 后 1-2 周是免费的"被加购转化"窗口期 — 此时砍预算 = 把别人塞到购物车的钱推出门。

---

## 🚫 5 个最常犯错误

1. **T-4 周还在临时备货** — FBA 入库要 2 周，T-2 周还没到货就完蛋
2. **PD 当天才加广告预算** — 没人在购物车里，CVR 不如有备而来
3. **价格突然降导致 Deal 被拒** — 必须提前 30 天涨价铺垫
4. **缺货还报 PD** — 用爆单速度死亡（排名永久受损）
5. **PD 后立刻砍预算** — 错过 view-through 收割期
