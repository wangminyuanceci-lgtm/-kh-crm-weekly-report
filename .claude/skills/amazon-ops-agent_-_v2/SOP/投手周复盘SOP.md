# 投手周复盘 SOP（30 分钟流程）

> 适用对象：单一品线广告投手 / 品线运营
> 使用前提：已有当周数据报告（跑过 `sales_ad_analysis.py` + `search_term_analysis.py`）
> 配套框架：[../框架/指定产品分析框架.md](../框架/指定产品分析框架.md) + [../框架/特征词分析框架.md](../框架/特征词分析框架.md)

## 🔥 4 步法（按顺序执行）

### 第 1 步：3 分钟看大盘 — 判断本周整体水位

打开 `广告分析报告-<品线>-0520.md`

只看 3 处：
1. **TL;DR**（顶部 3 句话结论）
2. **Layer 1 大盘表** — ACOS / TACOS / ROAS
3. **Layer 3 父ASIN 表的最后一列**（分析标签）

**3 秒判断**：

| 信号 | 解读 |
|---|---|
| 大盘 TACOS 跟上周比 ±2pp 内 | ✅ 整体稳，进入下钻流程 |
| TACOS 突涨 >3pp | ⚠️ Layer 3 找涨在哪个父ASIN |
| Top 3 主力父ASIN 标签都"健康型" | ✅ 主力没问题 |
| 出现新的 "🚨 优化重点型" | 🔴 这是本周重点 |
| 出现新的 "广告依赖型"（TACOS 异常高）| ⚠️ 待 4 周趋势确认 |

### 第 2 步：5 分钟锁定本周"必动的" 1-3 个父ASIN

在 Layer 3 父ASIN 表里只找两类：

**A. 标签 = "🚨 优化重点型"**
直接看 Layer 4 子ASIN 拆解段。

**B. 性价比标 "⚠️ 烧多卖少"** 且 Spend 占品线 ≥10%
（占比小的烧多卖少不痛，按"花得多才值得动"原则不管）

**经验法则**：通常每周只有 1-3 个父ASIN 真需要动。其他**全部维持现状**。

### 第 3 步：10 分钟下钻定具体动作

对锁定的父ASIN，看 Layer 4 子ASIN 拆解表：

#### 字段优先级
1. **"Spend / 组占"列** — 只动占组 ≥ 10% 的 SKU（小钱不动）
2. **库存** — 警惕 "库存 0 但本周有销售" = 数据失真，**不要停广告**
3. **店CVR vs 广CVR** — 差距 >10pp 说明关键词错配
4. **CTR** — 低于品线均 0.5 倍 = 主图/关键词不匹配，listing 问题

#### 子SKU 决策树

| 信号 | 动作 |
|---|---|
| ACOS 高 + Click ≥ 30 + **非策略性 Campaign** | 降出价 30% 观察 2 周 |
| 店CVR > 广CVR 差 >10pp + Spend 占组大 | 加 Negative（关键词错配）|
| CTR < 品线均 0.5 倍 | listing 主图/标题改写（找设计/listing 团队）|
| 库存 < 30天 + 是主力 SKU | 推动补货 + 暂停广告防引流空 |
| 真断货（库存=0 + 销售=0 + Amazon 后台 Available=0）| 暂停广告 + 补货 |
| ACOS 高 + Spend 占组 < 5% | 知悉不动（钱小） |

### 第 4 步：10 分钟看关键词 + 广告位 → 形成最终清单

#### 关键词分析（`关键词分析-<品线>.md`）

打开 xlsx 的 `按搜索词聚合` sheet，按 "诊断状态" 过滤：

**🚨 效果较差**：通常 3-25 个
- 看 Top 5 烧钱 → 这些已经在 Layer 4 报告里下钻过 Campaign
- 加 Negative 或重做关键词

**⭐ 表现优秀**：通常 5-15 个
- 筛 "Orders ≥ 3 + ACOS < 大盘 + 当前无 Exact 投放" 的词
- 提取为独立 Exact Campaign（**$5/天小预算测试 2 周**，不要直接大额投）

#### 广告位（`广告位分析-<品线>.md`）

**98% 情况下不动**。只看 1 个数：
- SB Other ROAS 是否 > 10？
  - 是 + 已经稳定 4 周 → 考虑给 SB 加预算（注意是预算，不是 Bid Modifier）
  - 否 → 不动

```

### 🔴 P0 立即做（库存/断货类）
- [ ] 紧急补货 SKU: XXX（库存 X件/X天）

### 🟡 P1 本周做（高 ROI 优化）

- [ ] 加 Phrase Negative "XXX" 到 N 个 Campaign
- [ ] 主图/标题改写 SKU XXX（CTR X% 远低于均）
- [ ] 新建 Exact Campaign：[关键词]（小预算 $5/天测试）

### 🟢 P2 看 4 周趋势再决定
- [ ] SKU XXX ACOS 异常但单周判断不可靠（标记观察）

### 🚫 知悉不动
- [ ] SKU XXX（占组 < 5% 钱小不动）
- [ ] Campaign XXX（策略性投放，不评估）
```

## ⚡ 投手决策速查表（贴墙用）

### 何时该动 ✅

| 信号 | 动作 |
|---|---|
| 父ASIN 标 🚨 优化重点 + Spend 占品线 ≥5% | 下钻子ASIN |
| 单 SKU ACOS > 大盘 +30pp + Click ≥ 1/CVR + 数据稳定 | 降出价 30% |
| 店CVR > 广CVR 差 >10pp | 加 Negative |
| ⭐ 表现优秀词 + 无 Exact | 提取扩词（小预算）|
| 主力 SKU 库存 < 60d | 推动补货 |
| 真断货（3 重验证） | 暂停广告 + 补货 |
| SB Other ROAS > 10 持续 4 周 | 加投 |

### 何时不该动 ❌

| 信号 | 原因 |
|---|---|
| 父ASIN 标 "健康型" | 不动 |
| 单 SKU 周 Spend < $30 | 钱小不值得 |
| 单周 0 单 + 新品/高单价 | 等 4 周 |
| 库存表 0 但本周有销售 | 数据失真，先核查 Amazon 后台 |
| 跨品线 Campaign（MZ-Catch Them all 等）| 不归你管 |
| Campaign 命名含"引流/拓词/捡漏" | 策略性投放，不看单周 ACOS |
| 单周 ACOS 高但 4 周趋势波动 | 等趋势确认稳定差再动 |

## 🚫 投手 5 个最常踩的坑

1. **看单周数据直接 Pause Campaign** — 之前我们试过 exercise mats SD（单周 ACOS 182% 看似该停，但 9 周看 ACOS 6.5%-340% 大幅波动 + 投入低，应该不动）
2. **用 Click-only 看 SD 报告** — SD 是展示广告，**必须用 V&C 字段**，否则 ACOS 看起来虚高（如 153% 实际 25%）
3. **库存 0 立即停广告** — 库存数据经常失真（在途未计入 / 数据时点错位 / 导出 bug），**先核查 Amazon 后台真实 Available**
4. **看到高 ACOS 就该降** — 策略性投放（品类渗透 / 拓词 / 配套销售）短期 ROAS 不是评估指标
5. **把跨品线 Campaign 归到本品线诊断** — MZ-Catch Them all 等"全店捡漏"应该排除，整体看才公平

## 📅 标准周流程示例

### 周一 9:00 - 10:30（90 分钟完整复盘）

| 时间 | 任务 | 输出 |
|---|---|---|
| 9:00-9:05 | 跑数据 | 当周 0520/ 目录 |
| 9:05-9:35 | 完成 4 步法复盘 | 本周动作清单 |
| 9:35-10:00 | Amazon 后台执行 BulkSheet | 调整生效 |
| 10:00-10:30 | 执行非 BulkSheet 动作（Coupon/Listing brief 等）| 工单/委托 |

### 周中（无需每天看数据）

- 只在出现重大异常（如 Sales 突降）时才看
- 单周判断不可靠原则：日常不要"修修补补"

### 下周一开始

- 复盘上周动作效果：上周标 P0/P1 的动作是否带来预期改善？
- 失败的复盘记录（如 "降 Bid 后 ROAS 反而降了" → 下次不这么做）

## ❓ 常见问题

**Q: 我应该每天看报告吗？**
A: 不需要。日常优化用 Amazon 后台实时数据。本 SOP 是周复盘用，定期"系统性看"，不是日常"修修补补"。

**Q: 看到大量 🚨 效果较差，全部 Pause 吗？**
A: 不要！按"花得多才值得动"原则，只动占组 ≥10% Spend 的；且 Pause 前要看 4 周趋势。

**Q: 报告说"待 4 周趋势确认"是什么意思？**
A: 单周数据不可靠（之前 exercise mats SD 单周 182% / 9 周 6.5%-340% 大幅波动的教训）。看 Amazon 后台手动跑 4 周 Campaign 趋势图确认稳定差才动。

**Q: 跨品线 Campaign 不归我管，那归谁？**
A: 全店运营 / BD 主管。看 `0520/跨品线Campaign诊断-0520.md`。

**Q: 主图改写报告说要做，但我不是设计？**
A: 把报告里"CTR 0.15% 远低于品线均"的 SKU 名单 + 改写建议甩给 Listing 团队/设计部，让他们做。你只负责广告参数。
