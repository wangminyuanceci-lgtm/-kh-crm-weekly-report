# actions/ — 动作清单(追踪闭环)

每周一份文件:`YYYY-WNN.md`(ISO 周编号),与 `reports/YYYY-WNN/` 对应。

## 文件用途

把"诊断 → 建议"变成"执行 → 验证 → 沉淀"的闭环:

```
周一 /weekly-review 自动创建 actions/YYYY-WNN.md
   ├─ 写入本周诊断产出的动作 (A001...)
   ├─ 运营平时下钻产出的动作自动追加
   ↓
运营线下操作后 → 手动改 [待执行] → [已执行] + 填实际执行
   ↓
下周 /weekly-review 自动回读 → 对比指标 → 填复查结论
   ├─ 达预期 → [已验证] 关闭
   ├─ 未达 → 延期 / 加新动作
   └─ 反向 → 立即回滚 + 新动作
```

## 文件规范

完整规范见 [SOP/动作追踪SOP.md](../SOP/动作追踪SOP.md) + [模板/动作清单模板.md](../模板/动作清单模板.md)。

要点:
- ID:A001 起本周递增(跨周不连续)
- 状态:`[待执行]` / `[已执行]` / `[已验证]` / `[已逾期]` / `[已取消]`
- 每条必有:优先级 / 品线-父ASIN / 动作类型 / 根因 / 预期效果 / 执行人 / 计划完成 / 复查节点

## 文件命名

- 周报对应:`reports/2026-W21/` ↔ `actions/2026-W21.md`
- 用 ISO 周编号,周一为一周开始

## 历史动作

- 保留全部历史周(不删)
- 跨周复查:下周自动读上周;3 周前的复查结论也在文件里能查

## 运营怎么用

```
周一收到周报通知 → 打开 actions/本周.md 看 P0/P1 动作
   ↓
周一-周三:逐条到 Amazon 后台执行
   ↓
执行完一条:打开文件,把 `状态: [ ] 已执行` 改成 `[x] 已执行`,
            填 `实际执行: 日期 + 简短备注`
   ↓
下周一周报 A 段会自动复查 → 看结论
```

## 不放入此目录的

- ❌ 临时草稿
- ❌ 完整诊断报告(那些在 reports/)
- ❌ 非本周的杂项笔记
