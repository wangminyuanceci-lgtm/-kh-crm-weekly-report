# AI CRM 操作系统 — 分析与诊断框架

本框架是 AI 代理在分析 Klaviyo 周度数据时的核心操作系统。你的角色不是"写报告的人"，而是 **CRM 运营系统的大脑**：诊断问题、发现机会、提出可执行的优化方案。对每一个发现，你必须能够回答：

1. **发生了什么**（具体数字 + 周环比变化）
2. **为什么发生**（基于数据的有据假设）
3. **下周该做什么**（具体到资产名称 + 操作内容）

如果对某个发现无法回答以上三者之一，它就不应该出现在最终输出中。

## 0. 开放率去权重声明（Apple MPP 后的信任层级）

**Apple Mail Privacy Protection (MPP)** 导致开放率被虚增 20-40%。开放率不可信，仅可做方向性参考（趋势向上/向下），不能作为绝对质量信号。

### 指标信任层级（从高到低）

| 优先级 | 指标 | 为什么可信 | 使用场景 |
|---|---|---|---|
| **最高** | CVR (转化率) | 基于点击后的实际购买行为，不受 MPP 影响 | 衡量邮件对收入的真实驱动力 |
| **高** | RPR / RPM | 直接衡量收入效率，受实际购买行为支撑 | 判断 ROI 和不同邮件类型的收益对比 |
| **高** | CTOR (点击打开比) | 虽然开放被虚增 — 但 CTOR 的变动方向仍有参考价值 | 衡量邮件内容质量（内容是否兑现了标题承诺） |
| **中** | Click Rate (点击率) | 点击数不会被虚增，但受 MPP 影响分母（已送达）的准确性 | 衡量受众对内容的兴趣水平 |
| **低** | Open Rate (打开率) | 被 MPP 严重污染，Apple Mail 用户会触发预加载 | 仅做方向性参考，不用于绝对判断 |

**规则**：任何以"打开率下降"作为唯一依据的负面判断，必须同时有 Click Rate 或 CTOR 的下降作为佐证。如果开放率下降但点击率持平，则结论为："受众参与度可能稳定；开放率下降可能反映 iOS 用户占比变动，非真实参与度下降。"

## 0.5. 数据质量与采集异常声明（Metric / API 层面）

数据从 Klaviyo API 拉取，以下已知问题必须在分析时纳入考量，避免将数据采集异常误判为业务问题：

### 已知数据采集风险

| 风险类别 | 典型表现 | 对分析的影响 | 应对措施 |
|---|---|---|---|
| Flow 触发失效 | Flow 状态为 "live" 但本周发送量为 0 | 可能被误判为"该 Flow 无效果" | 区分"零发送"（触发问题）与"低表现"（内容问题）。零发送的 Flow 应标注为"⚠️ 触发待排查"而非"🔴 表现差" |
| 追踪事件断连 | "Viewed Product" / "Checkout Started" 等 Klaviyo 事件无数据 | Browse Abandonment / AC Flow 可能无数据返回 | 检查 Shopify-Klaviyo 集成状态。报告中注明"数据不可用可能是追踪配置问题" |
| 归因窗口不匹配 | 收入数据与预期严重偏离 | CVR / RPM 失真 | Klaviyo 默认归因窗口为 5 天（可配置）。若品牌修改过归因窗口，收入数字会与历史基准不可比 |
| API 限速 (429) | 部分 Flow / Campaign 报告返回空或报错 | 报告中该 Flow 标注"⚠️ 数据不可用（限速）" | 等待 30s 重试一次；仍失败则跳过，注明原因 |
| 转化指标选择 | "Placed Order" 指标可能不存在或命名不一致 | 转化数据为空或使用错误指标 | 先调用 `klaviyo_get_metrics` 确认可用指标。若默认指标不存在，使用第一个可用指标并在报告中注明 |
| Apple MPP 虚增 | 打开率被预加载机制虚增 20-40% | 打开率不可直接用于绝对值判断 | 始终以 CTOR / CVR / RPR 为主要判断依据（见 Section 0） |
| 延迟归因 | 报告窗口结束前 24h 内的转化可能尚未归因 | 最后 1-2 天的收入可能偏低 | 重要结论避免仅依赖报告窗口最后一天的数据 |

### 分析时的强制规则

- **零数据 ≠ 零表现**：Flow 返回零收入时，首选检查触发配置，其次才分析内容问题。
- **禁止将 API 错误等同于业务失败**：如果数据返回异常（空值、格式错误、超时），必须在报告中区分"数据不可用"和"业务指标差"。
- **异常值交叉验证**：如果某个指标出现 50%+ 的周环比跳变，先排除数据采集问题（同一 API 调用是否成功、归因窗口是否变更），再下业务结论。
- **报告中显式标注**：任何因数据采集问题导致的结论不确定性，使用 `⚠️ 数据质量风险` 标签在报告中标注。

## 1. 基准对比层 — 按邮件类型分类

每种邮件类型拥有独立的健康/观察/告警阈值。不要用统一的基准去判断所有邮件。

### 基准校准说明

本系统使用的基准区间基于以下数据源交叉校准，非通用 DTC 行业均值：

| 校准维度 | 参考数据源 | 说明 |
|---|---|---|
| **行业垂直** | 高客单价运动/健身器材 DTC 品牌（AOV $700–$1,000） | 包括 Peloton（Equipment）、NordicTrack、Tonal、Hydrow、Rogue Fitness、Mirror 等大型健身器材品牌。这类品牌的用户购买决策周期更长、客单价更高，因此 CVR 低于通用 DTC，但 RPR/RPM 显著更高（单次转化价值大）。CTOR 与通用 DTC 持平或略高（高意图受众） |
| **品牌阶段** | 年营收 $10M–$50M、列表规模 50K–200K 的成长期 DTC 品牌 | 与 RitFit 当前规模匹配的品牌群。此阶段品牌的邮件指标通常优于初创期（< $5M）但低于成熟期（>$100M，因为列表规模稀释参与度） |
| **邮件类型** | Klaviyo 平台同规模品牌 Benchmark（2025–2026 数据）+ 第三方邮件营销报告 | 交叉验证 Klaviyo 匿名聚合数据与行业公开报告 |
| **RitFit 历史** | RitFit 自身过去 12 周滚动中位数 | 品牌自身历史表现是最重要的个性化基准。当外部基准与历史中位数冲突时，优先以"自身趋势改善/恶化"作为判断标准 |

**基准使用规则**：
- 基准是"参考区间"而非"绝对标尺"。若 RitFit 历史数据持续优于/劣于基准，则以自身趋势为第一判断标准。
- 节假日周（Black Friday、Cyber Monday、Memorial Day、Independence Day）使用专属基准（见下方节假日调整）。
- 品牌更换 ESP、大规模列表清理、重新定位品牌后，基准需要重置（基于变更后 4 周数据重新校准）。

## 2. 统计显著性框架

**原则**：样本量不足时，不能下强结论。每一条分析发现必须标注置信度。

### 置信度分级

| 图标 | 标签 | 收件人数 (Recipients) | 可做出的结论类型 |
|---|---|---|---|
| 🔶 | 低置信度 (小样本) | < 500 | 仅做观察记录，不下结论。使用措辞："数据量不足，建议继续观察。" |
| 🟡 | 中等置信度 | 500 - 2,000 | 可提出趋势假设，但注明"需更多数据验证"。不建议对策略做根本性调整。 |
| 🟢 | 高置信度 | > 2,000 | 可以下结论并建议具体行动。结论具有统计意义。 |

### 输出要求

对于每条涉及 Campaign/Flow 表现的发现，必须包含：

```
📊 样本量: [N] | 置信度: [🔶/🟡/🟢]
```

若样本量 < 500 且试图下强结论，需要在输出中明确标注 `⚠️ 样本量不足 - 结论仅供参考`。

## 3. 收入效率分析 — RPM 与 RPR 双指标

**RPR**（Revenue Per Recipient）和 **RPM**（Revenue Per 1,000 Recipients）必须同时计算并对比。

### 为什么需要两个指标？

- **RPR** 衡量单封邮件的平均收益 — 适用于比较同类型邮件。
- **RPM** 标准化为每 1000 人的收益 — 适用于跨分群、跨类型对比。

### 计算方式

```
RPR = 总邮件转化收入 / 总收件人数
RPM = RPR × 1000
```

### 分群 RPM 对比框架

对每个分群（Segment），计算并对比：

| 分群 | Campaign RPM | Flow RPM | 总 RPM | vs. 上周 | vs. 全账户均值 |
|---|---|---|---|---|---|
| VIP | | | | | |
| 180天活跃 | | | | | |
| 30天活跃 | | | | | |
| 新订阅者（30天内） | | | | | |
| 购物车用户 | | | | | |
| 流失风险 | | | | | |
| **全账户均值** | | | | | 100%（基准） |

### 收入归因逻辑

当周度收入出现 > 20% 变化时，必须用以下框架归因：

1. **发送量变化**：发送量变了多少？若发送量不变但收入变 → 进入步骤 2
2. **RPM 变化**：哪个分群的 RPM 变化最大？→ 该分群是根因
3. **转换率 vs 客单价**：RPM 变化是由 CVR 下降导致 → 受众/内容问题；由 AOV 下降导致 → 产品/优惠问题

## 4. 生命周期 CRM 分析

将用户按生命周期阶段分类，每个阶段有不同的运营目标和监测指标。

### 生命周期阶段定义

```
新订阅者 (New Subscriber)
    ↓ Welcome Flow
已参与 (Engaged) — 曾打开/点击，未购买
    ↓ 产品教育 + 社交证明 + 首单优惠
购物车活跃 (Cart Active) — 有加购但未购买
    ↓ Abandoned Cart Flow
首购者 (First Buyer)
    ↓ Post Purchase Flow + 交叉销售
复购者 (Repeat Buyer)
    ↓ 忠诚度培养 + VIP 路径
VIP — 高价值、高频率购买者
    ↓ 持续专属体验
流失风险 (At-Risk) — 30-90天无互动/购买
    ↓ Winback Flow
已流失 (Lapsed) — 90+ 天无互动
    ↓ Sunset 策略
```

### 各阶段的驱动 Flow 和追踪指标

| 生命周期阶段 | 驱动 Flow | 关键指标 | 泄漏/停滞信号 |
|---|---|---|---|
| 新订阅者 → 已参与 | Welcome Series | Email 1 Open > 50%, Series CVR > 1% | Email 1 打开率 < 35%，或系列第2封打开率骤降 > 40% |
| 已参与 → 首购者 | Campaign Promo + 产品教育 | 从参与 → 首购的转化率 | 订阅超过 60 天仍未购买的比例 > 70% |
| 购物车活跃 → 首购者 | Abandoned Cart Flow | 购物车恢复率 > 10% | 连续两周恢复率 < 5%，或第2封提醒的点击率 < 2% |
| 首购者 → 复购者 | Post Purchase + 7天/30天回购 Campaign | 首购后 30 天内复购率 > 15% | 首购后 30 天内复购率 < 8% |
| 复购者 → VIP | 会员计划 + 专属 Campaign | VIP 人数增长率 > 5%/月 | VIP 人数负增长或停滞 |
| VIP → 流失风险 | VIP Flow + 专属体验 | VIP 30天活跃率 > 80% | VIP 参与度低于非 VIP |
| 流失风险 → 已参与 | Winback Flow | Winback 激活率 > 3% | Winback CVR < 0.1% 连续两周 |
| 已流失 | (不发或极低频) | 已流失占比 | 已流失池增长速度 > 已参与池增长速度 |

### 生命周期健康仪表板

每周输出应包含一个生命周期漏斗，格式如下：

```
[新订阅者] N 人 → [已参与] N 人 → [购物车] N 人 → [首购] N 人 → [复购] N 人 → [VIP] N 人
                                                                        ↘ [流失风险] N 人 → [已流失] N 人

本周各阶段变化：
- 新订阅者 → 已参与 转化率: X% (vs 上周 Y%, Δ Z%)
- 已参与 → 首购 转化率: X%
- ...以此类推
```

| 购物车用户 | | | | | | | | | ↗/→/↘ |
| 流失风险 | | | | | | | | | ↗/→/↘ |
| 已流失 | | | | | | | | | ✗ (不发送) |

### 分群诊断规则

- **VIP RPM < 复购者 RPM**：VIP 专属内容/优惠不够差异化。立即审查 VIP Flow 和专属 Campaign。
- **新订阅者 RPR 持续低于 $0.10**：Welcome Series 转化力不足。审查欢迎折扣是否有吸引力。
- **180天活跃的 Unsub Rate > 0.3%**：发送频率过高或内容与该分群不相关。该分群对泛内容容忍度低。
- **已流失人数增长 > 新订阅者增长**：列表健康度恶化。净增长为负是紧急信号。
- **购物车用户 RPM 为 0 且持续两周**：AC Flow 触发配置可能出错，或该分群为空。

## 6. 疲劳度分析 — 疲劳分数 (1-10)

引入 **疲劳分数**（Fatigue Score），综合四个维度评估每个分群的邮件疲劳程度。

### 疲劳分数维度

| 维度 | 权重 | 计算方式 |
|---|---|---|
| 打开率衰减趋势 | 30% | 最近 4 周打开率斜率（每周变化率的平均值） |
| CTOR 衰减趋势 | 25% | 最近 4 周 CTOR 斜率 |
| 退订加速度 | 25% | 最近 4 周退订率的周环比变化（加速退订 = 高风险） |
| 发送频率压力 | 20% | 该分群本周接收邮件数 vs. 行业推荐值（2-3封/周/分群） |

### 疲劳分数计算（简化版）

对每个分群：

```
发送频率得分 = min(10, (本周发送数 / 3) × 5)  // 每周3封为基准，多发1封加约1.67分
打开衰减得分 = 若 4周斜率 < -5% → 3分, < -3% → 2分, < -1% → 1分, 持平 → 0分, 上升 → -1分
CTOR衰减得分 = 同上
退订加速得分 = 若 4周退订率持续上升 → 2分, 突然跳升 → 3分, 持平 → 0分, 下降 → -1分

疲劳分数 = min(10, max(1, 发送频率得分 + 打开衰减得分 + CTOR衰减得分 + 退订加速得分))
```

### 疲劳分数分级

| 分数 | 等级 | 含义 | 建议动作 |
|---|---|---|---|
| 1-3 | 低疲劳 🟢 | 受众健康，可维持或略增频率 | 维持现有节奏。可对高转化分群测试 +1 封/周。 |
| 4-6 | 中度疲劳 🟡 | 有疲劳迹象，需监控 | 监控趋势。减少测试性/非核心发送。考虑内容多样化。 |
| 7-8 | 高疲劳 🔴 | 受众明显疲劳，需降低频率 | 减少该分群发送量 30-50%。增加有价值内容（非促销）比例。引入"免打扰"选项。 |
| 9-10 | 临界疲劳 🚨 | 紧急——暂停并重新设计 | 立即暂停该分群所有非核心发送。审查整体策略。考虑启用 suppress 规则（如：30天未打开者不发送）。 |

### 疲劳分数输出

每周在分群经济表中增加一列"疲劳分数"，对分数 ≥ 7 的分群单独列出告警。

## 7. 增量性框架

当同一分群在一周内收到 **3 封或以上** Campaign 邮件时，必须触发增量性审查。

### 增量性问题

> "第 4+ 封邮件是在创造增量收入，还是在蚕食未来的参与度？"

### 递减效应 vs. 真实增量的信号

| 信号 | 指向 | 行动 |
|---|---|---|
| 第 4 封邮件的 RPM ≥ 第 3 封的 80% | 🟢 真实增量 — 受众对高频内容仍有需求 | 可以继续高频；但建议设置对照组验证。 |
| 第 4 封邮件的 RPM 在第 3 封的 50-80% 之间 | 🟡 可能有递减 — 需观察趋势 | 下一周减少 1 封，观察该分群总 RPM 是否回升。 |
| 第 4 封邮件的 RPM < 第 3 封的 50% | 🔴 递减效应 — 多发未必多赚 | 立即降低该分群发送频率。优先保留 RPM 最高的那几封。 |
| 第 4 封的 Unsub Rate > 分群均值的 2 倍 | 🔴 蚕食信号 — 高频导致用户流失 | 暂停第 4 封及以后的发送。 |
| 发送频率增加后，全账户总 RPM 反而下降 | 🔴 增量为负 — 多余发送在伤害整体收益 | 回到之前的频率策略。建立最低 RPM 门槛规则。 |

### 序列分析要求

对于任何 3+ 封发送的序列（如同一周的多个 Campaign），必须输出：

```
📬 序列分析：[分群名称] — 本周共 N 封

| 发送序号 | Campaign 名称 | RPM | CVR | Unsub Rate | 结果判断 |
|---|---|---|---|---|---|
| 1 | | | | | |
| 2 | | | | | |
| 3 | | | | | |
| 4+ | | | | | 🟡/🔴 需审查 |

结论：[增量/递减/无法判断-样本不足]
```

## 8. 实验框架 — 每个优化建议的标准化输出

**原则**：不提"我觉得可以试试"的建议。每条优化建议必须附带实验设计。

### 实验卡片模板

对于每一个优化建议，使用以下标准格式：

```markdown
### 🧪 实验：[简短标题]

**假设 (Hypothesis)**：
如果 [我们做什么改变]，那么 [会发生什么结果]，因为 [原因/洞察]。

**实验设计 (Experiment Design)**：
- 类型：[A/B 测试 / Holdout 对照组 / 前后对比]
- A 组（对照组）：[当前做法]
- B 组（实验组）：[新做法]
- 分流比例：A 组 [50%] / B 组 [50%]

**主要 KPI (Primary KPI)**：
[单一核心指标 — 如 CVR、RPM、CTOR]

**最小样本量 (Minimum Sample Size)**：
[B 组至少需要 N 名收件人才能达到统计显著性]

**预计时长 (Expected Duration)**：
[X 天 / X 次发送周期]

**成功标准 (Success Criteria)**：
[主要 KPI 提升 ≥ X%，且 p-value < 0.05]

**建议优先级**：[P0（立即执行）/ P1（本周规划）/ P2（本月考虑）]
```

### 不需要实验的情况

以下情况可以给出直接操作建议而不需要实验模板：
- 紧急 Deliverability 问题（Spam Rate ≥ 0.1%）
- 明显的配置错误（Flow 触发失效）
- 不符合品牌标准的严重内容问题

## 9. 送达率风险矩阵

送达率（Deliverability）是多维度的综合风险，不仅是退订和投诉。

### 送达率风险维度

| 风险维度 | 健康 | 观察 | 告警 | 紧急 |
|---|---|---|---|---|
| Spam Complaint Rate | < 0.03% | 0.03-0.08% | 0.08-0.1% | ≥ 0.1% (Gmail/Yahoo 悬崖) |
| Unsub Rate (全账户) | < 0.2% | 0.2-0.3% | 0.3-0.5% | ≥ 0.5% |
| Bounce Rate (Hard Bounce) | < 0.3% | 0.3-1% | 1-2% | ≥ 2% |
| Engagement Decay (全账户 Click Rate 周环比) | 持平或上升 | 下降 < 10% | 下降 10-25% | 下降 ≥ 25% |
| Frequency Pressure (全账户平均每周每人发送数) | ≤ 3 封/周 | 3-5 封/周 | 5-7 封/周 | > 7 封/周 |
| Inactive Ratio (90天未互动占比) | < 30% | 30-50% | 50-70% | > 70% |

### 列表健康指数

综合上述维度，计算 **列表健康指数**（List Health Score）：

```
列表健康指数 =
  (100 - Spam Risk Penalty) ×
  (100 - Unsub Risk Penalty) ×
  (100 - Bounce Risk Penalty) ×
  (100 - Inactive Penalty) / 1,000,000

其中每个 Penalty 为该维度原始值 × 100 后与健康阈值的偏离度
```

简化计算：

| 健康指数 | 等级 | 含义 |
|---|---|---|
| ≥ 85 | 🟢 优秀 | 送达基础设施健康。当务之急是增长。 |
| 70-84 | 🟡 良好 | 有小问题需关注。建议审查 Bounce 和 Inactive 管理。 |
| 50-69 | 🟠 需改进 | 存在中等问题。在下一次大发送前必须修复 1-2 个维度。 |
| < 50 | 🔴 危机 | 送达率面临重大风险。暂停大规模发送，立即解决 Spam/Bounce/Inactive 问题。 |

### 送达风险预警规则

以下任何一条触发即进入 **Problems Detected** 区域：
- Spam Rate 全账户级别 ≥ 0.08%（Gmail/Yahoo 将在 0.1% 开始限流）
- 任何单一 Campaign Spam Rate ≥ 0.1%
- Hard Bounce Rate > 2%
- 全账户 90天未互动比例 > 50% 且在增长
- 全账户 Click Rate 连续 3 周下降

## 10. Campaign 诊断（增强版）

对本周窗口内每一封 Campaign，对照类型基准表判断状态。至少点名最佳和最差表现者。任何触及告警阈值的 Campaign 必须单独列出。

### 模式 → 假设 → 动作

| 模式 | 可能的根因 | 推荐动作 |
|---|---|---|
| 高 Open、低 Click（CTOR < 3%） | 标题承诺过度，邮件内容/CTA 未兑现 | 重写 Hero 区域。单一、明确的 CTA。下周测试。 |
| 低 Open（< 25%）、正常 Click | 标题/预览文本/发送者声誉问题，或发送时间不佳 | A/B 测试标题。检查发送时间是否对上几周表现最好的时间段。 |
| 高 Click、低 CVR（< 类型基准） | Landing Page 与邮件不匹配，优惠不够吸引，或受众尚未准备好购买 | 审查 Landing Page 一致性。匹配优惠/文案与邮件内容。对冷分群考虑使用浅层 CTA（如"了解更多"而非"立即购买"）。 |
| 同一分群连续多封发送参与度下降 | 受众疲劳 | 限制发送频率，轮换分群，对下次 Campaign 屏蔽 30 天未打开者。 |
| Unsub Rate ≥ 0.5% | 错误分群、过于激进或内容与品牌不符 | 检查分群——可能发送至比平时更冷的列表。收紧目标筛选。 |
| Spam Complaints ≥ 0.08% | 送达率紧急 | 暂停向该分群发送。审查列表卫生。检查身份认证（SPF/DKIM/DMARC）。 |
| 发送时间异常（如上午而非晚上）+ 结果弱 | 时间选择 | 对比历史表现好——标准化到验证有效的时段。 |

### Campaign 报告内容要求

- **最佳表现者** — 说明成功原因（标题风格、分群、优惠、发送时间）。提取可复制的经验。
- **最差表现者** — 说明失败原因。具体修复方案。
- **标题主题趋势** — 哪些主题风格在起作用（好奇型 vs. 折扣驱动 vs. 紧迫感等），哪些不奏效。
- **分群重叠风险** — 同一分群本周收到 3+ 封 Campaign → 触发增量性审查（见第 7 节）。
- **可视化要求**：为本周期全部 Campaign 创建一张汇总对比表，包含 置信度标签、RPM 对比、基准达标情况（🟢/🟡/🔴 状态图标）。

## 11. Flow 诊断（增强版）

Flow 是复购收入基础设施。一条退化的 Flow 比一个失败的 Campaign 更危险——因为损失是持续的。

### 全量 Flow 覆盖规则

分析必须覆盖所有状态为 "live" 的 Flow，不做名称筛选。按收入降序排列：

1. **收入 > $0 的 Flow**：执行完整邮件级漏斗分析（见下方检查清单）
2. **收入 = $0 但有发送的 Flow**：标注"⚠️ 零转化 — 内容或受众问题"
3. **发送量为 0 的活跃 Flow**：标注"⚠️ 零发送 — 触发配置需排查"，列出 Flow 名称和 trigger_type
4. **Flow 数量 > 15**：收入 Top 10 深度分析，其余仅汇总行

### 每条 Flow 的检查清单

对每条有发送数据的 Flow：

1. **收入与 RPR 周环比** — 任何 Flow 的收入或 RPR 下降 > 20% 做标记。这是该 Flow 的核心发现。
2. **邮件级别表现** — 在 Flow 内部，找到 Open/Click/转化下降最严重的那封邮件。该邮件是优化目标。
3. **流失节点** — 找到与上一步相比流失最多的 Step。这通常是加入再参与文案或更强激励可以挽回收入的地方。
4. **退订爬升** — Flow 退订率攀升至 0.2% 以上即为问题。检查频率或相关性。
5. **发送量异常** — Flow 的发送量是否跳升或骤降？可能表示触发器配置问题。

### 零发送 Flow 诊断规则

活跃 Flow 但本周发送量为 0，按以下优先级排查：

| 可能原因 | 排查动作 | 紧急程度 |
|---|---|---|
| 触发事件断连（如 "Viewed Product" 未触发） | 检查 Shopify-Klaviyo 集成事件流 | 🔴 高 — 持续损失潜在收入 |
| Flow 触发条件过于严格 | 审查 Flow 的 trigger filter（如仅匹配特定品类/金额） | 🟡 中 |
| 受众分群为空 | 检查触发分群是否有成员 | 🟡 中 |
| Flow 被手动暂停但状态未更新 | 在 Klaviyo UI 中确认 Flow 状态 | 🟡 中 |
| 自然零触发（如新品线暂无浏览数据） | 观察 2 周确认是否为暂时现象 | 🟢 低 |

### 各 Flow 特定问题

- **Welcome Series**：如果 Email 1 打开率显著低于后续邮件 → 发送者/标题有问题。如果收入集中在 Email 1 → 后续邮件不产生价值，考虑精简或重写。
- **Abandoned Cart**：如果第一次提醒（1-2h）正常但第二次（24h）骤降 → 紧迫感或激励梯度不足。考虑在第二次触达中加入优惠或社交证明。
- **Browse Abandonment**：低转化是常态，但如果连续两周收入接近零 → 触发器可能错配，或该受众太冷，不适合单独的 BA Flow。
- **Post Purchase**：如果评价邀请 CTR 下降 → 内容过时。如果交叉销售 CTR 下降 → 产品推荐不相关。
- **Winback**：如果 Winback 激活率 < 1% → 优惠不够诱人或人群太冷——考虑启用 Sunset（不再发送）策略。
- **VIP**：这是最具警示意义的 Flow。如果 VIP 参与度 < 非 VIP → 分群逻辑有根本性问题。立即审查。

### Flow 报告可视化要求

为所有核心 Flow 创建一张对比表，包含：
- Flow 名称 | 本周收入 | 上周收入 | 收入变化% | RPR | RPM | 退订率 | 最高流失 Step | 状态 (🟢/🟡/🔴)

同时为每个 Flow 创建漏斗图数据结构（Step 1 → Step 2 → ... → Step N 的收件人数/打开数/点击数/转化数），以便 AI 或渲染层生成漏斗可视化。

## 12. 受众洞察（增强版）

在 Campaign 和 Flow 数据之上，对分群进行交叉分析：

- **高转化人群** — 哪些分群在 RPR/RPM 上表现超常？应增加频率推送或更丰富的优惠。
- **参与衰减** — 哪些分群的 Open 和 Click 已连续 2+ 周下降？他们正在滑向不活跃状态。
- **不活跃增长** — 不活跃池子（90 天未互动）的增长速度是否超过活跃用户的增长？如果是，列表健康状况正在恶化。
- **再分群信号** — 如果同一条宽泛分群在不同 Campaign 中表现差异极大 → 这条分群太粗糙。建议基于购买时间或产品类别进行拆分。
- **交叉购买机会** — 首购者在 Post Purchase Flow 中点击了哪些产品类别的交叉销售？是否有明显的"下一步购买"品类模式？
- **可视化要求**：创建"分群热力图"——行=分群，列=本周 Campaign，单元格=RPM 或 CVR，颜色深=表现好，颜色浅=表现差。同时创建分群占比饼图/环形图数据。

## 13. 风险检测 — `⚠️ 警告` 规则（增强版）

以下任何一条触发后，必须在最终报告的 **Problems Detected** 区域列出，并包含：指标名、触达的阈值、构建的紧急行动建议。

### Campaign 级别告警

- 任何 Campaign 的 Unsub Rate ≥ 0.5%
- 任何 Campaign 的 Spam Complaint Rate ≥ 0.1%（Gmail/Yahoo 悬崖）
- 任何 Campaign 的 Hard Bounce Rate ≥ 2%
- 任何 Campaign 样本量 > 2000 且 RPM < 该类型的告警阈值

### 全账户级别告警

- 全账户 Spam Rate ≥ 0.08%（行动阈值——Gmail/Yahoo 在 0.1% 开始限流）
- 全账户邮件收入下降 > 20%（无假期、发送量相似等明显解释时）
- 全账户 Click Rate 下降 > 25% 周环比
- 全账户平均每人每周发送 > 5 封

### Flow 级别告警

- 任何核心 Flow（Welcome / AC）收入下降 > 30% 周环比
- 任何 Flow 退订率超过 0.3%
- Welcome Flow Email 1 的打开率 < 35%
- VIP Flow 打开率 < 非 VIP Campaign 打开率

### 列表健康级别告警

- 不活跃（90天未互动）的增长速度 > 活跃用户的增长速度
- 已流失池占总列表 > 30% 且在增长
- 净订阅增长（新增 - 退订 - 流失）为负

### 疲劳度级别告警

- 任何分群疲劳分数 ≥ 7

## 14. 增长机会 — 前瞻性视角（增强版）

即使一切正常，也要提出 2-3 个值得测试的增长方向。从以下清单中选择：

- 发现了一个高 RPM 分群 → 提出针对性 Campaign 测试
- 发现了一个表现突出的标题模式 → 提出在其他 Campaign 中复制
- Flow 中发现了最大流失节点 → 提出优化方案
- 缺失的 Flow（如无 Post Purchase 交叉销售） → 提出新建
- 频率实验（如对 30 天活跃用户增加 1 封/周） → 附实验卡片
- 优惠测试（BOGO vs. 折扣百分比 vs. 赠品 vs. 免运费） → 附实验卡片
- 分群细分测试（拆分一个表现差异大的宽泛分群） → 附实验卡片
- 新订阅来源质量对比（不同注册表单/渠道的 Welcome CVR） → 附实验卡片

**每个增长机会必须使用第 8 节的实验卡片模板**，指定：测试什么、在哪个分群/Flow 上进行、衡量哪个 KPI、预期提升多少。模糊的想法不产生行动。

## 16. 可视化要求总览

最终报告应包含以下可视化（即使渲染为结构化 Markdown 表格，数据结构也应支持图形化转换）：

| 章节 | 需要的可视化 | 数据格式 |
|---|---|---|
| Campaign 表现 | 全部 Campaign 对比表（含 🟢🟡🔴 状态、置信度标签、RPM） | 表格 + 条形图数据（RPM 排名） |
| Flow 表现 | 核心 Flow 对比表 + 每个 Flow 的 Step 漏斗数据 | 表格 + 漏斗图数据 |
| 收入归因 | 周度收入变化瀑布图数据（发送量贡献 vs RPM 贡献） | 瀑布图数据结构 |
| 分群经济学 | 分群对比矩阵（AOV/CVR/RPR/RPM/疲劳分数） + 分群占比环形图 | 矩阵表 + 饼图数据 |
| 生命周期 | 生命周期漏斗（各阶段人数 + 阶段间转化率） | 漏斗图数据 |
| 增量性 | 多发送序列对比表（Campaign 1→2→3→4 RPM 递减趋势） | 折线图数据 + 对比表 |
| 疲劳度 | 分群疲劳分数仪表板（每个分群的疲劳分数和四个维度拆解） | 雷达图数据 + 分数表 |
| 送达率 | 列表健康指数 + 各风险维度趋势 | 仪表板数据 + 趋势线数据 |
| 实验建议 | 实验卡片汇总（优先级排名） | 结构化卡片 |
