# 飞书发布工作流

本文描述如何将完成的周报发布到飞书云文档。根据当前会话可用的集成方式，选择对应的模式执行。

## 命名规范

命名规则按品牌区分：

| 报告品牌 | 文档标题格式 | 示例 |
|---|---|---|
| RitFit（默认） | `RitFit EDM Weekly Performance Report - W{WW}` | `RitFit EDM Weekly Performance Report - W22` |
| KH | `KH CRM Weekly Performance Report - W{WW}` | `KH CRM Weekly Performance Report - W26` |

- 其中 `{WW}` 是报告窗口**结束日期**所在的 ISO 周数（两位数）。
- 示例（KH）：报告窗口 2026-06-22 至 2026-06-28，结束日期 2026-06-28 属于 ISO Week 26，文档标题为 `KH CRM Weekly Performance Report - W26`。
- 计算 ISO 周数的方法：确定结束日期所在年份的 ISO 周定义（周一为一周起始，第一周包含当年第一个周四）。可借助日期工具或在线计算确认。
- 格式固定：前缀按品牌选择（见上表），分隔用 ` - `（空格-连字符-空格），周数前缀大写 `W` 紧跟两位数（如 `W26`）。文档标题与文档内 H1 必须完全一致。

如果目标文件夹中已存在同名文档，追加 `(v2)`、`(v3)` 等后缀，禁止静默覆盖已有周报。

## 模式 A —— 飞书 MCP 可用

检查当前会话中是否有以下工具（名称可能因集成而异）：

- `mcp__feishu_*` / `mcp__lark_*`
- 包含 `create_document`、`create_doc`、`docx_create`、`docs_create_document` 的工具
- 包含 `list_folder`、`drive_list`、`find_folder` 的工具
- 包含 `get_doc_content`、`get_document`、`docs_get`、`read_document` 的工具（用于验证步骤）

### 步骤

1.  **创建文档。**
    - 使用飞书文档创建工具，传入固定文件夹 token `CoEefbPNflZk7AdD8kncIGRDnvd` 和按命名规范生成的标题。
    - 记录返回的 `document_id` 和分享链接。

2.  **写入报告内容。**
    - 优先使用 Markdown 模式写入（如果工具支持），我们的报告使用 Markdown 编写。
    - 如果仅支持块级 API，则按顶级章节（以 `## ` 标题为分界）拆分为独立文本块写入，确保文档生成后可生成大纲。
    - 表格（`| col | col |`）在飞书文档中以 Markdown 导入时能正确渲染。

3.  **验证发布结果（文档完整性检查）。**
    - 读取刚创建的文档内容或大纲。
    - 确认以下 3 大章节全部存在：
        1.  重点数据表现（含 Executive Dashboard、排行榜、趋势图表）
        2.  数据诊断分析（含 Campaign 诊断、Flow 诊断、生命周期洞察、风险预警）
        3.  优化建议和方向（含实验建议、下周行动清单）
    - 如果任一章节缺失，重新写入完整报告。

4.  **记录分享链接。**
    - 在用户回复中附上文档的飞书分享链接。

5.  **权限（可选）。**
    - 如果用户之前指定过默认分享权限（如"租户内所有人可查看"），则应用该权限。否则保留 API 默认设置。

## 模式 C —— 无飞书集成（降级方案）

当前环境没有飞书 MCP 工具和 API 凭据时，执行以下操作：

1.  **禁止编造飞书链接。**
2.  将完整报告 Markdown 保存到工作目录下的本地文件，文件名遵循命名规范：`RitFit EDM Weekly Performance Report - WXX.md`。
3.  向用户回复以下内容：
    - 本地文件路径
    - 明确说明："当前会话未连接飞书集成，报告已保存至本地。如需自动发布，请添加飞书 MCP 服务或提供应用凭据。"
    - 可在聊天中直接展示执行摘要（Executive Summary），让用户立即获取核心信息。

报告本身绝不能因发布失败而丢失。报告撰写是核心产出，飞书发布只是传输手段。

## 输出语言规范

所有报告内容（文档正文、章节标题、分析文字）必须使用简体中文。以下内容可保留英文/数字形式：

- 品牌名称（如 RITFIT Sports、Klaviyo）
- 指标缩写（CTOR、CVR、RPR、RPM 等）
- 货币符号（$）
- 数字与百分比

## 输出质量规则

撰写和发布报告时，必须遵守以下格式规则：

- **禁止输出 Markdown 格式残留**：不允许出现 `**` 加粗标记、`---` 分隔线、`` ` `` 行内代码标记等原始 Markdown 语法。如果工具要求 Markdown 输入则在源文件中保留，但最终渲染到飞书文档时不得出现这些标记。
- **数字格式**：
    - 金额：`$12,345`（美元符号 + 千分位逗号）
    - 百分比：`42.5%`（始终保留一位小数，如 `0.35%`、`100.0%`）
    - 比率指标：`0.35%` 或 `$1.25` 等形式
- **禁止重复货币符号**：不得出现 `$$` 或 `$ $`。
- **禁止输出原始 JSON 或未处理数据**：所有数据必须在写入前完成格式化，报告中不得出现 JSON 结构、API 响应原文或任何未加工的数据字段。
- 表格中的数字同样适用上述规则。

## 错误处理

- **文档创建成功但内容写入失败** → 如可行则删除空文档（避免遗留空白文档），重试一次。仍失败则降级到模式 C，本地文件名保留文档标题。
- **限流 / 429 错误** → 等待 30 秒后重试一次。仍失败则降级到模式 C。
- **认证过期** → 向用户明确提示，不得静默跳过。

始终在用户回复中说明使用了哪种模式、是否发生重试或降级。透明告知胜过静默部分失败。
